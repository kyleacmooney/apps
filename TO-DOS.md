## Add In-App Species Research via Edge Function + Claude API - 2026-03-08 17:15

- **Add "Research Species" button to Plants UI** - Let users trigger Claude-powered species research directly from the web app instead of requiring a separate Claude.ai session. **Problem:** Currently, populating `species_profiles` requires the user to open a Claude.ai conversation with the Supabase MCP connector and follow `docs/plant-care-instructions.md`. This is friction — a button in the plant detail sheet that says "Research this species" would be much more seamless. **Files:** `src/pages/Plants.tsx` (PlantDetailSheet already shows a "No species research yet" prompt — add a button there), `supabase/functions/` (new edge function needed), `docs/plant-care-instructions.md` (reference for what data to populate). **Solution:** Create a new edge function (e.g., `research-species`) that: (1) receives a species name, (2) calls the Anthropic API with a prompt derived from `plant-care-instructions.md` to research the species, (3) upserts the result into `species_profiles`, (4) optionally triggers `proxy-image-upload` for the image. The SPA calls this via `supabase.functions.invoke('research-species', { body: { species_common_name } })`. The edge function holds `ANTHROPIC_API_KEY` as a secret (`supabase secrets set`). Auth: verify the Supabase JWT from the Authorization header to prevent abuse. Consider streaming the response for UX, or just show a loading spinner since the research prompt is a single structured output. The Claude.ai workflow via `plant-care-instructions.md` should remain as an alternative for more complex/conversational research.
